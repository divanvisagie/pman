Manual First

pman is an opinionated framework for a project workflow that maximizes context reuse, note management, and collaboration between humans, AI agents, and Unix tooling. It draws from the Unix philosophy: small, deterministic commands with predictable outputs and a strict hierarchical file structure. The repository is the source of truth for this workflow, and this page is the entry point.

The reason it exists is simple: most workflows fail in the boring parts. Naming, structure, and bookkeeping drift over time. pman makes those parts deterministic so projects stay searchable, notes stay aligned, and people (or agents) stay in sync. The CLI only automates the deterministic parts; the workflow itself can be executed manually, by an AI agent, or even with notebooks and sticky notes.

Set Up Your Workspace

Create a strict hierarchy under ~/src. This becomes the single, shared context for both humans and AI agents.

1. Create ~/src and place your projects in a reverse-domain tree.

2. Create ~/src/Notes and follow PARA (Projects, Areas, Resources, Archives).

3. Copy the AGENTS file into ~/src/AGENTS.md as the workflow contract.

Connect Notes Everywhere

Notes are the shared context. Keep them close to every repo by symlinking or cloning.

1. Symlink ~/src/Notes into each project (recommended), or clone it directly.

2. Ensure each repo has an Area note and reference it from project notes.

3. Use the registry in Notes/Projects to track active work.

Run the Workflow

The workflow can be executed manually or automated with the CLI. The CLI only handles deterministic steps like creating and archiving project notes.
