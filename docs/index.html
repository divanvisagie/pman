Manual First

pman is an opinionated framework for a project workflow that maximizes context reuse, note management, and collaboration between humans, AI agents, and Unix tooling. It draws from the Unix philosophy: small, deterministic commands with predictable outputs and a strict hierarchical file structure. The repository is the source of truth for this workflow, and the manual is the point of entry.

The reason it exists is simple: most workflows fail in the boring parts. Naming, structure, and bookkeeping drift over time. pman makes those parts deterministic so projects stay searchable, notes stay aligned, and people (or agents) stay in sync. The CLI only automates the deterministic parts; the workflow itself can be executed manually, by an AI agent, or even with notebooks and sticky notes.

Why It Works

The workflow removes ambiguity in naming, structure, and bookkeeping. It keeps humans and agents aligned by enforcing deterministic project creation and a single source of truth for notes.

How It Works

1. Store projects in a reverse-domain tree under ~/src.

2. Keep notes in ~/src/Notes using PARA.

3. Create project notes in Notes/Projects with a PROJ id.

4. Archive completed projects in Notes/Archives/Projects.

5. Keep the AGENTS file up to date as the contract for behavior.

Using the CLI

The CLI covers deterministic steps like creating and archiving project notes. It is intentionally small and predictable. See the CLI reference for commands and flags.
